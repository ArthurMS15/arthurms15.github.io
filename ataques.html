*Recursos/ativos que ele pode potencialmente afetar:

Os ativos afetados vão de dispositivos individuais (computadores pessoais e smartphones), até redes inteiras, sistemas de controle industrial, infraestruturas críticas e sistemas governamentais;

Sabendo que qualquer setor que tenha qualquer conexão a rede implementada em sua organização pode estar sujeito a ataques, qualquer requisito de segurança pode ser afetado também, mas poderia ser comumente mais atrelado a confidencialidade, integridade e privacidade, já que esses estão mais atrelados a manipulação dos dados.

Malware:

Se refere a todo o tipo de software malicioso (vírus, cavalos de Troia, worms e ransomware). Os atacantes usam para roubar informações confidenciais, criptografar arquivos importantes ou transformar sistemas em bots para distribuir spam ou lançar ataques DDoS. Existem várias técnicas: phishing, engenharia social, exploração de vulnerabilidades em sistemas desatualizados e outros. Uma vez que o usuário clica em um link ou baixa um arquivo malicioso, o malware é instalado no dispositivo sem o conhecimento. Pegando como referência o parágrafo anterior ele então acaba roubando informações pessoais, gravar a atividade do usuário, instalar outros malwares, criptografar dados e exigir um resgate, controlar o dispositivo infectado, entre outros.

Phishing:

Envio de mensagens falsas por e-mail ou mensagem de texto fingindo ser uma fonte confiável. O objetivo é enganar e levar a pessoa a clicar em um link, fazer download ou revelar informações pessoais e confidenciais (roubo de identidade, fraude financeira ou invasão de sistemas).
Com o uso de e-mails falsos, mensagens ou chamadas telefônicas se passando por entidades legítimas. Com técnicas de engenharia social servindo para enganar os usuários, como promessas falsas de prêmios ou ameaças de ações legais.


Por meio da engenharia social, em que o criminoso busca manipular o usuário para que ele revele informações pessoais ou confidenciais. Com técnicas de persuasão ou links para sites falsos que se parecem com os sites legítimos, mas que foram criados para roubar informações do usuário.

Denial of Service (negação de serviço):

Sobrecarrega um sistema com tráfego de rede excessivo (falso) ou solicitações de serviço para que ele se torne indisponível. Enviando uma grande quantidade de pacotes de dados para o alvo, sobrecarregando recursos e tornando-o indisponível para usuários legítimos, impedindo o acesso ao serviço.


Através de uma rede de computadores infectados com malware (botnet), que é controlada pelo atacante e utilizada para enviar o tráfego malicioso para o alvo. Ou por meio da amplificação de tráfego: o atacante envia pacotes para servidores de terceiros que respondem com uma quantidade ainda maior de tráfego para o alvo.


Com requisições falsas ou usando técnicas de sobrecarga de conexões. O alvo tenta processar todos os pedidos recebidos, mas acaba ficando sobrecarregado e não consegue atender novas solicitações de usuários legítimos. Causando prejuízos financeiros, perda de dados ou danos à reputação da organização afetada.

Man in the Middle:

Intercepta e altera a comunicação entre duas partes que acreditam estar se comunicando diretamente entre si. Envolve um invasor no meio de uma comunicação para ver e potencialmente modificar os dados trocados.
Através de redes Wi-Fi públicas não seguras, servidores DNS comprometidos, ou através da infecção de um dispositivo ou roteador com malware.


O invasor pode usar técnicas de phishing, como e-mails de spear phishing ou mensagens de texto falsas, para enganar uma das partes e fazê-la baixar um software malicioso.

SQL Injection:

Usado para explorar vulnerabilidades em sistemas que utilizam bancos de dados e que não aplicam corretamente as práticas de segurança para prevenir a injeção de código malicioso. O invasor injeta o código em uma aplicação web através de campos de entrada passados para um servidor de banco de dados.


Através da exploração de vulnerabilidades em aplicações web que interagem com bancos de dados. Utilizando-se de softwares automatizados que tentam inserir códigos maliciosos em campos de entrada de formulários, como caixas de texto, formulários de login e outros.


O invasor aproveita das vulnerabilidades para inserir comandos maliciosos no banco de dados. Esses comandos podem ser usados para obter informações sensíveis, como senhas e dados de cartão de crédito, ou para modificar e excluir informações no banco de dados.

Cross-site Scripting (XSS):

Consiste em inserir código malicioso em uma página web, com o objetivo de comprometer a segurança do usuário que acessa essa página. O invasor insere códigos maliciosos em páginas web que serão executados pelo navegador do usuário, permitindo assim que o invasor execute ações em nome do usuário, como roubar informações confidenciais ou comprometer a integridade do sistema.

Exploração de vulnerabilidades presentes em páginas web, especialmente aquelas que permitem a inserção de dados pelos usuários. Inserindo em campos de entrada de dados, como formulários ou caixas de pesquisa, ou em URLs, que, quando acessados pelo usuário, executam o código malicioso.



Quando o usuário acessa uma página que contém código malicioso injetado, o navegador executa como se fosse da página legítima. Existindo dois tipos de XSS: armazenado, onde o código é armazenado no servidor da página web e exibido em todas as visitas subsequentes, ou do tipo refletido, onde o código é refletido de volta para o usuário através de uma solicitação enviada a partir de um site legítimo.

Drive-by Downloads:

O ataque ocorre quando o usuário visita um site comprometido que contém código malicioso, que é executado automaticamente sem conhecimento ou interação do usuário. Instalando um malware e podendo comprometer outros dispositivos da rede.

Pode ser operacionalizado através da exploração de vulnerabilidades em softwares de navegadores da web, plugins e outros aplicativos utilizados para acessar a internet.



Quando o usuário visita um site comprometido, o código malicioso é baixado e executado automaticamente no navegador da web, explorando vulnerabilidades no software. O ataque também pode ser conduzido através de anúncios maliciosos exibidos de sites legítimos (malvertising).

Password attacks:

O invasor tenta obter acesso não autorizado a uma conta protegida por senha. O objetivo é obter a senha para acessar a conta e executar ações maliciosas.



O ataque de senha pode ser operacionalizado de várias maneiras, como: Brute force (tentativa e erro), Dictionary attack (usa um dicionário de senhas comuns), Phishing (engana o usuário para que ele revele), Keylogging (software que registra todas as teclas digitadas).
O ataque funciona explorando vulnerabilidades do processo de autenticação. O ataque pode ser devastador, porque senhas tendem a ser a primeira linha de defesa. É importante que as senhas sejam fortes e exclusivas.

Engenharia social:

Manipulação psicológica para obter informações confidenciais ou acesso a sistemas e recursos, enganando as vítimas. O ataque se aproveita da confiança das pessoas, sendo realizados por meio de telefonemas, e-mails, mensagens de texto, redes sociais, entre outras formas de comunicação.
Existem diversas técnicas para conseguir: Phishing (uso de sites falso ou e-mail que se passa por uma empresa), Pretexting (se passar por outra pessoa ou instituição), Baiting (oferecer uma recompensa para a vítima).


Começa com a coleta de informações sobre a vítima para a manipulação. Em seguida, o atacante seleciona a técnica de engenharia social que melhor se adapta à vítima e cria um cenário crível para fazer com que a vítima revele informações ou execute ações que possam comprometer a segurança.

Cryptojacking:

Uso da capacidade de processamento do computador da vítima para minerar criptomoedas sem permissão. O objetivo é gerar lucro para o atacante.



Realizado por meio de um script malicioso instalado em um site, um e-mail de phishing ou um anexo de e-mail infectado. Quando o usuário é infectado, o script é executado e começa a usar o computador da vítima para minerar criptomoedas.
O ataque começa com extensões, envio de um e-mail ou anuncio com um link para um site malicioso (contém o script). Quando a vítima visita, o script é executado e usa o computador da vítima para minerar criptomoedas em segundo plano.

DNS Spoofing:

O invasor manipula o sistema de nome de domínio (DNS) de uma rede, substituindo um endereço IP legítimo por um endereço falso. Redirecionando o tráfego de rede, para então roubar informações confidenciais do usuário.




Envolve a exploração de vulnerabilidades em sistemas de DNS mal configurados ou desatualizados, a interceptação do tráfego de rede em trânsito ou a instalação de malware em um dispositivo na rede.




O ataque começa com o invasor enviando uma solicitação DNS falsificada para um servidor DNS legítimo. Quando o usuário na rede faz uma solicitação para acessar um site, o servidor DNS malicioso responde com o endereço IP falso. O ataque é eficaz quando o invasor tem acesso ao gateway da rede.

Botnets:

São redes de dispositivos infectados com malware, sendo controlados remotamente pelo atacante. O objetivo é usar essas redes para realizar uma série de atividades maliciosas (spam, negação de serviço e roubo de informações pessoais).
Os atacantes geralmente criam botnets infectando dispositivos por meio de downloads de software maliciosos ou outras vulnerabilidades. Uma vez que os dispositivos são infectados, eles se conectam a um servidor, permitindo que o atacante os controle.
Começa com a infecção de um dispositivo com malware (com ele vindo de várias fontes). Depois o malware se conecta a um servidor de comando e controle controlado pelo atacante. A partir daí, o atacante pode enviar comandos para a rede botnet.

Zero-day exploits:

Explora vulnerabilidades em sistemas de computador e software que ainda não são conhecidas por desenvolvedores, baseado em falhas recém-descobertas e não corrigidas.






Realizados por hackers experientes que buscam explorar falhas em sistemas específicos. Eles podem ser realizados através de diferentes métodos, incluindo engenharia reversa, exploração de brechas de seguranç, interceptação de dados e outros.




Hackers procuram por falhas de segurança em sistemas e exploram as falhas para obter acesso não autorizado. Como essas vulnerabilidades ainda não são conhecidas, os sistemas não possuem nenhuma proteção contra esses ataques. quando a vulnerabilidade é descoberta, geralmente é tarde demais para evitar os danos causados pelo ataque.
